아래의 동영상을 보고 느낀점을 정리합니다.

최신 데이터 인프라 이해하기 - YouTube

제품 포지셔닝

먼저 AWS, GCP, Azure 등은 클라우드 벤더로서 AIDA가 제공하는 가치와 조금은 다른 제품이라 생각합니다.

아래 설명을 드리겠지만 cuve는 amundsen이라는 data catalog 제품군에 가깜고, c3는 AWS EMR과 가깝습니다.

따라서 우리가 추구하는 것이 Big 3 클라우드 벤더가 지향하는 것이 아닌 최종 고객이 될 데이터 분석가들의 니즈에 맞추어야 할 것입니다.

따라서 위 동영상에 나오는 최신 데이터 관련 SaaS를 통해 최신 트렌드와 고객의 니즈를 파악하고 우리가 가진 cuve와 c3로 어떠한 가치를 제공할 수 있는지 고민해야 합니다.

Ingestion

고객은 데이터 분석을 위해 데이터 소스로 부터 data warehouse로 보관하고 싶어 합니다.

Fivetran은 이러한 문제를 풀어주는 솔루셥입니다.

고객의 데이터는 물론 Salesforce, SAP, Github, Stripe 등의 SaaS 서비스로 부터 Databricks, Snowflake, Big Query와 같은 data warehouse 제폼으로 데이터를 주입해 줍니다.

현재 cuve에는 cuve writer 및 stream, api 등이 있지만 사용자들이 직접 구현해야 하고 사전에 협의해야 합니다.

이미 잘 알려져 있는 사내의 데이터소스 예를들어, nelo, DBIP(SQL Gateway)등으로 부터 cuve라는 data warehouse로의 주입을 self-service 할 수 있어야 합니다.

Discovery & Governence

고객은 데이터를 찾고, 접근을 제어하고, GDPR과 같은 complience를 위한 audit, lineage, 테이터 테스트, 문서화 등의 기능을 바탕으로한 가버넌스 도구를 원합니다.

Collibra는 이러한 문제를 풀어주는 솔루션 입니다.

Amundsen과 Datahub는 오픈소스 솔루션입니다.

고객은 RDBMS, Object storage, datawarehouse, hdfs와 같은 다양한 소스로 부터 수집한 metadata를 통해 데이터를 조회합니다.

cuve는 이 카테고리의 제품군 이라 생각합니다.

Data transform/analysis

고객은 여러 데이터 소스를 denormalize 하여 분석에 필요한 정제된 데이터를 만들고 싶어 합니다.

Snowflake, databricks는 이러한 문제를 해결합니다.

Spark는 jupyter 등의 오픈소스와 함께 이러한 문제를 해결합니다.

고객은 warehouse에 있는 데이터를 SQL, Python, R 등을 사용하여 분석에 필요한 데이터로 가공합니다. 데이터 가공은 pipeline을 통해 이벤트나 특정 주기에 따라 수행합니다. dbt를 사용하여 SQL 구문이나 비지니스 로직을 코드 기반으로 재사용 및 협업합니다.

cquery는 이 카테고리의 제품군에 속합니다. c3는 이 카테고리의 제품군에 필요한 연산 환경을 제공합니다.

Reporting

고객은 여러 지표를 바탕으로 구성원들과 공유할 수 있는 리포팅을 만들고 싶어 합니다.

Colab, hex는 이러한 문제를 해결합니다. (looker, mode 등도 있음)

c3는 이 카테고리의 제품군에 필요한 연산 환경을 제공합니다.

ML

고객은 정제된 데이터를 바탕으로 ML 학습 및 serving을 하고 싶어 합니다.

데이터 스텍

서비스를 운영하면서 발생하는 여러 데이터를 한 곳에 모아 분석을 해야 합니다. 이렇게 분석한 결과를 서비스 지표로 삼아 개선이 필요한 부분을 찾아내거나 비용 및 성과를 측정합니다.
