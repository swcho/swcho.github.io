Preconstruct

preconstruct

preconstruct 설정은 Preconstruct라는 도구를 위한 것입니다.
Preconstruct는 모노레포나 패키지를 쉽게 빌드하고 배포할 수 있게 도와주는 툴입니다.

여기서 "entrypoints"는 패키지의 진입점(엔트리포인트) 파일들을 지정합니다.
즉, 이 패키지를 사용할 때 외부에서 직접 접근할 수 있는 주요 파일 목록입니다.

예시 설명:

  • "index.ts": 패키지의 기본 엔트리포인트
  • "fiber/index.ts": fiber 하위 모듈의 엔트리포인트
  • "vite.ts": 별도의 엔트리포인트

Preconstruct는 이 파일들을 바탕으로 각각의 엔트리포인트에 대해 빌드 결과물을 만들어줍니다.

정리:
이 설정은 패키지를 여러 엔트리포인트로 빌드하고, 각각을 외부에서 import/export 할 수 있게 해줍니다.

참고:
Preconstruct를 쓰면 TypeScript, 여러 엔트리포인트, 번들링 등 복잡한 빌드 설정을 간단하게 관리할 수 있습니다.

Babel 커스텀 플러그인

이 코드는 Babel 플러그인으로, React 컴포넌트의 JSX와 훅 호출을 변환하여 react-three-editor에서 사용할 수 있도록 메타데이터와 편집 기능을 추가합니다.

주요 기능 요약

  1. JSX 요소 변환

    • JSXOpeningElement 방문자에서 JSX 요소를 분석해,
      • 특정 조건(예: 소문자로 시작하는 태그, 대문자로 시작하는 컴포넌트, 네임스페이스 사용 등)에 따라
      • 요소 이름을 Editable로 바꾸거나, component 속성을 추가하거나, 네임스페이스를 조정합니다.
    • 각 JSX 요소에 __source(TRACE_ID) 속성을 추가해,
      • 파일명, 라인, 컬럼, 모듈명, 컴포넌트명, 엘리먼트명 등 위치 정보를 객체로 기록합니다.

  2. 훅 변환

    • useFrame → useEditorFrame,
      useUpdate → useEditorUpdate로 자동 변환합니다.
    • 호출 위치의 부모 컴포넌트명을 첫 번째 인자로 추가합니다.
      (컴포넌트별로 호출 순서를 구분하기 위해 카운터도 붙임)

  3. import 변환

    • Program의 exit에서 지정된 import를 파일 상단에 추가합니다.
    • ImportDeclaration에서 특정 import 경로를 옵션에 따라 치환합니다.

  4. 유틸 함수

    • findParentReactComponent: 부모 React 컴포넌트 선언을 찾음(대문자로 시작하는 함수/변수).
    • getName: LVal에서 이름 추출.
    • makeTrace: JSX의 위치 및 메타데이터 객체 생성.

목적

  • JSX 요소와 훅에 에디터용 메타데이터를 삽입하여,
    런타임에서 어떤 컴포넌트/엘리먼트가 어디서 생성됐는지 추적하고,
    에디터에서 편집 가능하게 만듭니다.

예시

<MyComponent />

→

<Editable component={MyComponent} __source={{ fileName, lineNumber, ... }} />

useFrame(fn);

→

useEditorFrame('MyComponent:0', fn);

이 플러그인은 react-three-editor에서 코드 위치 추적, 편집, 디버깅, 컴포넌트 식별 등에 활용됩니다.

Program hook

Program

이 코드는 Babel 플러그인의 visitor 패턴에서 Program 노드의 exit 시점에 호출됩니다.

  • Babel AST 트래버설에서 Program은 소스 코드 전체를 감싸는 최상위 노드입니다.
  • exit는 Babel이 전체 AST를 탐색(traverse)한 후, 즉 모든 하위 노드 방문이 끝난 뒤에 호출됩니다.
  • 이 시점에서 코드를 수정하면, 파일의 맨 위에 import 구문을 추가하거나 전체 구조를 바꿀 수 있습니다.

예시 흐름

  1. Babel이 소스 파일을 파싱해서 AST를 만듭니다.
  2. 모든 노드를 방문합니다.
  3. 마지막에 Program.exit가 호출됩니다.
  4. 여기서 import 구문을 AST에 추가합니다.

Gotcha

  • enter는 AST 탐색 시작 시점, exit는 끝난 후 시점입니다.

  • import 구문을 맨 위에 추가하고 싶을 때는 exit에서 처리하는 것이 일반적입니다

JSXOpeningElement hook

JSXOpeningElement(path, state)

이 코드는 Babel 플러그인에서 JSX 요소를 변환하는 트랜스포머 함수입니다. React 코드에서 JSX 태그를 분석하고, 특정 조건에 따라 태그 이름을 바꾸거나 속성을 추가합니다. 주요 목적은 "editable" 기능을 적용하고, 추적(trace) 정보를 삽입하는 것입니다.

주요 동작 단계

  1. 초기 검사

    • 노드에 위치 정보(loc)가 없거나 이미 __source 속성이 있으면 아무 작업도 하지 않습니다.

  2. 부모 컴포넌트 이름 추출

    • findParentReactComponent로 현재 JSX가 속한 컴포넌트의 이름을 찾습니다.

  3. JSX 태그 이름 분석 및 변환

    • 소문자로 시작하는 태그(HTML 태그 등):
      • isEditableElement가 true를 반환하면, 태그를 editable.태그명으로 바꿉니다.
    • 대문자로 시작하는 태그(React 컴포넌트):
      • isEditableElement가 true를 반환하면, component 속성을 추가하고 태그명을 Editable로 바꿉니다.
    • 네임스페이스가 있는 태그(예: Foo.Bar):
      • 네임스페이스가 editable이 아니고, 조건에 따라 component 속성을 추가하고 태그명을 Editable로 바꿉니다.

  4. 파일 이름 변수 삽입

    • 파일 이름을 추적하기 위해, 파일 이름을 담은 변수를 프로그램 최상단에 한 번만 삽입합니다.

  5. 추적(trace) 속성 추가

    • JSX 요소에 추적용 속성(TRACE_ID)을 추가합니다. 이 속성에는 파일명, 위치, 컴포넌트명, 엘리먼트명 등이 담깁니다.

예시

변환 전:

<div />
<MyComponent />
<Foo.Bar />

변환 후(조건에 따라):

<editable.div __trace={...} />
<Editable component={MyComponent} __trace={...} />
<Editable component={Foo.Bar} __trace={...} />

주의할 점(Gotchas)

  • isEditableElement 함수는 외부에서 주입되므로, 실제 동작은 옵션에 따라 달라집니다.
  • 파일 이름 변수는 한 번만 삽입됩니다.
  • JSX 네임스페이스 처리 시, editable 네임스페이스는 건너뜁니다.
